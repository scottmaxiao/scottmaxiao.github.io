IOS-Category

Objective-C提供了一种与众不同的方式——Category，可以动态的为已经存在的类添加新的行为。这样可以保证类的原始设计规模较小，功能增加时再逐步扩展。使用Category对类进行扩展时，不需要访问其源代码，也不需要创建子类。Category使用简单的方式，实现了类的相关方法的模块化，把不同的类方法分配到不同的分类文件中

创建Category

在Xcode中File->New->File然后选择Objective-C File。

点击下一步

填写"File"名称，选择"File Type"为"Category",选择"Class"。

点击下一步,选择要生成的文件目录。这样就建立了一个Category类。

Category类文件命名为"Class+File"。生成的文件包含"UIColor+ColorWithHexValue.h"和"UIColor+ColorWithHexValue.m"。

如下例:

UIColor+ColorWithHexValue.h



#import 

@interface UIColor (ColorWithHexValue)

/*!
 * @method 通过16进制计算颜色
 * @abstract
 * @discussion
 * @param 16机制
 * @result 颜色对象
 */
+ (UIColor *)gap_colorFromHexRGB:(NSString *)inColorString;

@end

UIColor+ColorWithHexValue.m



#import "UIColor+ColorWithHexValue.h"

@implementation UIColor (ColorWithHexValue)

/*!
 * @method 通过16进制计算颜色
 * @abstract
 * @discussion
 * @param 16机制
 * @result 颜色对象
 */
+ (UIColor *)gap_colorFromHexRGB:(NSString *)inColorString{
  return nil;
}

注意:

  • Category可以重载原始类的方法，但不推荐这么做，这么做的后果是你再也不能访问原来的方法。如果确实要重载，正确的选择是创建子类。
  • Category可以访问原始类的实例变量，但不能添加变量，如果想添加变量，可以考虑通过继承创建子类。

调用

调用Category类非常方便，只需要引入Category类，然后直接调用方法即可。


#import "UIColor+ColorWithHexValue.h"

[UIColor gap_colorFromHexRGB:@"f3f3f3"];

使用场景

Category的使用场景：

  1. 当你在定义类的时候，在某些情况下（例如需求变更），你可能想要为其中的某个或几个类中添加方法。
  2. 一个类中包含了许多不同的方法需要实现，而这些方法需要不同团队的成员实现
  3. 当你在使用基础类库中的类时，你可能希望这些类实现一些你需要的方法。

遇到以上这些需求，Category可以帮助你解决问题。
